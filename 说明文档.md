# 分学科教师需求填写系统 - 说明文档

## 一、项目规划

### 1.1 项目概述
面向学校教研场景的分学科教师需求填写与展示系统。教师可在线填写个人教研需求（三大分类共13个细分标签），提交后所有教师的需求以卡片形式公开展示，支持数据持久化存储。

### 1.2 技术选型
- **后端**：Python Flask
- **数据库**：SQLite
- **前端**：原生 HTML/CSS/JS（响应式布局）
- **运行环境**：Windows / Python 3.x

### 1.3 核心功能
1. 教师需求表单（姓名、学科、三大分类复选框+文本描述）
2. 需求提交与数据持久化
3. 需求展示区（卡片形式，按分类分组）
4. 删除已提交需求（含确认弹窗）
5. 响应式布局（PC + 移动端）

## 二、实施方案

### 2.1 数据库设计
- **submissions 表**：id, name, subject, created_at
- **submission_items 表**：id, submission_id, category, tag, description

### 2.2 API 设计
- `GET /api/needs` — 获取所有需求
- `POST /api/needs` — 提交新需求
- `DELETE /api/needs/<submission_id>` — 删除需求

### 2.3 目录结构
```
├── app.py              # Flask 后端
├── templates/
│   └── index.html      # 前端页面
├── needs.db            # SQLite 数据库（自动生成）
└── 说明文档.md          # 项目文档
```
